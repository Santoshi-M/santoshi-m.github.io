1 Logistic Regression ▼

Regression model that outputs a probability.

Probability ensured by the sigmoid function (logistic function class). S-shaped curve, range 0–1.

$$f(x) = \frac{1}{1 + e^{-x}}$$
Log-Odds (z)

\( z = b + w_1x_1 + w_2x_2 + \cdots + w_nx_n \)  (bias + weights · inputs)

$$\hat{y} = \frac{1}{1 + e^{-z}}$$
Why z is called "log-odds" — derivation
①\( 1 + e^{-z} = \dfrac{1}{\hat{y}} \)
②\( e^{-z} = \dfrac{1-\hat{y}}{\hat{y}} \)
③\( -z = \log\!\left(\dfrac{1-\hat{y}}{\hat{y}}\right) \)
④\( z = \log\!\left(\dfrac{\hat{y}}{1-\hat{y}}\right) \) — log-odds
Log-Loss (Cross-Entropy)

Used over squared loss because output is not linear. Near \(z=0\), small changes cause large swings in \(\hat{y}\).

$$\text{log-loss} = -\frac{1}{N}\sum_{i=1}^{N}\left[y\log\hat{y} + (1-y)\log(1-\hat{y})\right]$$
L2 Regularization
$$\text{Loss} + \lambda \sum w_i^2 \qquad (\lambda = \text{regularization rate})$$
  • Pushes weights toward zero but not exactly zero (square term).
  • Near-zero weights: diminished effect — e.g., \(0.1 \to 0.01\) vs L1's \(0.1 \to 0.1\).
  • Balance learning rate & regularization rate (performance vs. complexity).
2 Decision Tree ▼
  • Leaf node → prediction
  • Non-leaf node → condition (split)
Conditions
  • Axis-aligned — single feature used.
  • Oblique — combination of features used.
Splitters

Algorithms that find the optimal split at each node.

Entropy

Measure of randomness. 0 entropy = all samples in one class.

$$H = -p\log p - (1-p)\log(1-p)$$
Information Gain
$$IG = H(D) - \frac{|T|}{|D|}\,H(T) - \frac{|F|}{|D|}\,H(F)$$

3 Decision Forest ▼
"Wisdom of the crowd"
Advantages
  • Easy to configure (fewer hyperparameters)
  • Natively handles numeric & categorical data (no preprocessing)
  • Robust to noisy data
  • Trains & infers faster than NNs on small datasets
Random Forest
  • Creates n independent decision trees.
  • Same # examples per tree, different subsets via bootstrapping.
  • Bagging = Bootstrap AGGregation
  • Feature sampling — each tree sees a random feature subset.
  • Individual trees are not pruned.
  • Does not overfit as more trees are added — accuracy saturates.
  • OOB evaluation — evaluated on Out-Of-Bag examples.
Gradient Boosting

\( F \) = strong model  ·  \( f \) = weak model

$$F_{i+1} = F_i - f_i$$

At step 0: \( F_0 = 0 \). Label for \( f_0 \) = error of \( F_0 \) = \( F_0 - y \). Then \( F_1 = F_0 - f_0 \).

With Shrinkage (learning rate \(\gamma\))
$$F_{i+1} = F_i - \gamma f_i$$

Slows learning, improves generalisation. Similar to learning rate in NNs.

W1 What is a Neural Network? ▼

A neuron takes an input (e.g. size) and outputs a prediction (e.g. price). This is a Rectified Linear Unit (ReLU).

A neural network stacks many neurons — inputs feed into a hidden layer which feeds into the output. The network is densely connected — every input connects to every hidden unit, and the model chooses what to learn.

Supervised Learning use cases
Network typeUsed for
Standard NNCTR prediction, tabular data
CNNImage recognition
RNNAudio, text sequences
HybridAutonomous driving
Why now?
  • Large NNs perform well on large data; small NNs ≈ traditional algos (SVM, log reg) on small data.
  • We have more data, better computation, and better algorithms.
  • Key algorithmic improvement: sigmoid → ReLU for faster training (no vanishing gradient).
W2 Logistic Regression & Loss Functions ▼
Binary Classification setup

\((x, y)\) where \(x \in \mathbb{R}^{n_x}\), \(y \in \{0,1\}\). \(m\) training examples: \(\{(x^{(1)}, y^{(1)}), \ldots, (x^{(m)}, y^{(m)})\}\)

\( X = \begin{bmatrix} \cdots \end{bmatrix}_{n \times m} \qquad Y = [y^{(1)}, y^{(2)}, \ldots, y^{(m)}]_{1 \times m} \)
Logistic Regression

\( \hat{y} = P(y=1 \mid x) \),   \(0 \leq \hat{y} \leq 1\)

\( \hat{y} \approx (wx + b) \) passed through sigmoid:   \( \sigma(z) = \dfrac{1}{1+e^{-z}} \)

  • \(\sigma(z) \to 1\) when \(z\) is large
  • \(\sigma(z) \to 0\) when \(z\) is very negative
Loss Function (single example)

MSE makes gradient descent non-convex, so log-loss is used instead:

$$L(\hat{y}, y) = -\left(y \log \hat{y} + (1-y)\log(1-\hat{y})\right)$$
  • When \(y=0\): want \(-\log(1-\hat{y})\) small → \(\hat{y}\) should be small (close to 0)
  • When \(y=1\): want \(-\log\hat{y}\) small → \(\hat{y}\) should be large (close to 1)
Cost Function (all examples)
$$J(w,b) = \frac{1}{m}\sum_{i=1}^{m} L(\hat{y}^{(i)}, y^{(i)})$$

Cost function = average of all loss functions applied to each training example.

LR Cost Function — probabilistic view

\( P(y \mid x) = \hat{y}^y (1-\hat{y})^{(1-y)} \)

$$\log P(y \mid x) = y\log\hat{y} + (1-y)\log(1-\hat{y}) = -L(\hat{y}, y)$$
W2 Gradient Descent & Derivatives ▼
Gradient Descent

Goal: find the global minimum of cost function \(J(w,b)\) — that's why cost functions need to be convex.

$$w := w - \alpha \frac{dJ(w)}{dw} \qquad b := b - \alpha \frac{dJ(w,b)}{db}$$
(\(\alpha\) = learning rate)
Derivatives

Derivative = slope. \(\text{slope} = \dfrac{\Delta y}{\Delta x}\). Example: \(f(a) = a^2 \Rightarrow \dfrac{df(a)}{da} = 2a\).

Computation Graph

Breaks a function into sequential steps. Example: \(J(a,b,c) = 3(a+bc)\)

\( u = bc, \quad v = a+u, \quad J = 3v \)

\( \dfrac{dJ}{dv} = 3, \quad \dfrac{dJ}{da} = 3, \quad \dfrac{dJ}{db} = c \cdot 3, \quad \dfrac{dJ}{dc} = b \cdot 3 \)
Logistic Regression Gradient Descent
\( \dfrac{dL}{dz} = \hat{y} - y \)

\( \dfrac{dL}{dw_1} = \dfrac{dL}{dz} \cdot x_1, \quad \dfrac{dL}{dw_2} = \dfrac{dL}{dz} \cdot x_2, \quad \dfrac{dL}{db} = \dfrac{dL}{dz} \)
W2 Vectorization & Python Tips ▼
Vectorization

Instead of a for loop to compute \(z = w^T x + b\), use NumPy's np.dot — much faster (SIMD).

Vectorizing Logistic Regression
\( Z = w^T X + [b, b, \ldots, b] \)

\( dw = \dfrac{1}{m} \sum dx^T\,dz \qquad db = \dfrac{1}{m}\sum dz \)
Broadcasting

An \(m \times n\) matrix can be +/−/×/÷ by a \((1,n)\) or \((m,1)\) vector — Python copies it \(m\) or \(n\) times and does elementwise ops.

Tips for Python Vectors
  • Avoid rank-1 arrays (shape \(= (n,)\)) — always explicitly use \((n,1)\) or \((1,n)\).
  • Use assert a.shape == (m, n) to verify dimensions.
W3 Shallow Neural Networks ▼
Stacking multiple sigmoids
NN Representation

Notation: \(W^{[\ell]}_{\text{node}}\) — superscript = layer, subscript = node. Input layer not counted.

\( z^{[1]} = W^{[1]T}a^{[0]} + b^{[1]} \)

\( W^{T[1]} = 4\times3, \quad a^{[0]} = 3\times1, \quad b^{[1]} = 4\times1, \quad z^{[1]} = 4\times1 \)
Activation Functions
Sigmoid
\(\sigma(z) = \dfrac{1}{1+e^{-z}}\)

Rarely used now. Better for binary output layer.

Tanh
\(\tanh(z) = \dfrac{e^z - e^{-z}}{e^z + e^{-z}}\)

Centers mean of data. Almost always preferred over sigmoid.

ReLU (commonly used)
\(\max(0, z)\)

Faster learning — no vanishing gradient.

Leaky ReLU
\(\max(0.01z, z)\)

Small slope for \(z < 0\).

Derivatives of Activation Functions
Sigmoid: \(\dfrac{da}{dz} = a(1-a)\)    Tanh: \(\dfrac{da}{dz} = 1 - (\tanh z)^2\)

ReLU: \(g'(z) = \begin{cases}0 & z{<}0 \\ 1 & z{>}0\end{cases}\)    Leaky: \(g'(z) = \begin{cases}0.01 & z{<}0 \\ 1 & z{\geq}0\end{cases}\)
GD for NN — Forward & Backprop
Forward Propagation
\( Z^{[1]} = W^{[1]}A^{[0]} + B^{[1]} \)
\( A^{[1]} = g^{[1]}(Z^{[1]}) \)
\( Z^{[2]} = W^{[2]}A^{[1]} + B^{[2]} \)
\( A^{[2]} = \sigma(Z^{[2]}) \)
Backpropagation
\( dZ^{[2]} = A^{[2]} - Y \)
\( dW^{[2]} = \tfrac{1}{m}\,dZ^{[2]} A^{[1]T} \)
\( dZ^{[1]} = W^{[2]T} dZ^{[2]} \cdot g'^{[1]}(Z^{[1]}) \)
\( dW^{[1]} = \tfrac{1}{m}\,dZ^{[1]} X^T \)
Random Initialization
  • If all weights = 0, all nodes are symmetric → useless to have multiple nodes.
  • OK to initialize b as zeros. Initialize W randomly × 0.01.
W4 Deep Neural Networks ▼
Getting Dimensions Right
\( W^{[\ell]} \text{ shape: } (n^{[\ell]},\, n^{[\ell-1]}) \qquad b^{[\ell]} \text{ shape: } (n^{[\ell]},\, 1) \)

\( Z^{[\ell]},\, A^{[\ell]} \text{ shape: } (n^{[\ell]}, m) \qquad X \text{ shape: } (n^{[0]}, m) \)
Why Deep Representations?
  • Multiple hidden layers learn different aspects (edges → faces → people in vision).
  • Going deep is better than going wide — a shallow network needs exponentially more hidden units.
Building a DNN — Layer \(\ell\) block
Forward pass (cache z[ℓ])
Input: \(a^{[\ell-1]}\)
\( z^{[\ell]} = W^{[\ell]}a^{[\ell-1]} + b^{[\ell]} \)
Output: \(a^{[\ell]}\) · Cache: \(z^{[\ell]}, W^{[\ell]}, b^{[\ell]}\)
Backward pass
Input: \(da^{[\ell]}\)
\( dz^{[\ell]} \to dW^{[\ell]},\, db^{[\ell]} \)
Output: \(da^{[\ell-1]}\)

P1 Reflection Design Pattern ▼
Example: Ask LLM to write a draft email, then pass the draft in the next prompt to get feedback, and use the feedback to write the final email.
Key Properties
  • Reflection works better when additional input is given — e.g. passing error along with code.
  • Reflection is shown to provide better results.
  • Can use one model to do the task and another to critique.
Evaluating Reflection

Evaluate whether improvement is worth the extra cost:

  • Objective tasks — accuracy %
  • Subjective tasks — LLM as a Judge
LLM as a Judge works better for 1/0 answers. Instead of asking for a scale of 1–10, ask multiple 1/0 questions and sum them to get a score.
External Feedback

Spectrum: No reflection < With Reflection < Reflection with feedback

  • Feedbacks are facts provided alongside task output to the reflection LLM.
  • Feedbacks are usually code-generated — e.g. fact checker.
P2 Tool Use ▼

Letting LLMs call functions — their performance can improve significantly.

Example: LLM does not have access to current time, but you can pass code to return current time & LLM can use the tool to get the answer.
Tool Examples
  • Web search
  • DB query
  • Custom functions

LLMs can choose whether or not to use a certain tool. Tools are executed by us (not by the LLM).

Code Execution
  • Ask LLM to generate the code to create the tool — avoids hardcoding all possible needed tools.
  • Can also ask LLM to execute code, but run it in a sandbox environment to avoid catastrophic errors.
MCP — Model Context Protocol
  • MCP server is a wrapper for a tool.
  • MCP client is the app that is calling the tool.
Example:   Client = Claude Desktop  ·  Server = Slack
P3 Planning — Highly Autonomous Agents ▼
Planning with LLMs
  • LLM creates a plan to execute the task.
  • Use system prompt to ask LLM to create a step-by-step plan and return it in JSON format.
  • Parse the plan JSON to execute steps in downstream components.
Code Execution Alternative

Instead of JSON, ask LLM to return executable code. This method outperforms planning in JSON or plain text.

Latency & Cost Optimization
  • Latency — use a smaller model or a faster LLM provider.
  • Cost — benchmark cost of each component and decide whether it's needed.
  • Try multiple LLMs and use evals to choose.
  • Split the task into smaller steps.
  • Fine-tune a model on internal data.

4 Practical Tips for Building Agentic AI ▼
Evaluations (10–20 examples)
  • Eyeball example inputs & outputs to find common error types, write code to get them fixed, meanwhile tracking via an eval function.
  • For example: use regex or simple code to determine the correctness of the output.
  • For subjective error types (e.g. missing certain points in essay generation), use LLM as a Judge to measure against gold standard failing points.
Error Analysis
  • Examine traces to identify which step in the flow to focus efforts on.
  • Manually count up the errors of each component.
  • Build component-level evals.
Improving Components
  • Non-LLM components — tune hyperparameters (e.g. increasing number of search results).
  • LLM components can be improved by:
    • Improving prompts — few-shot prompting, more explicit examples
    • Trying new models — try multiple LLMs & use evals to choose
    • Splitting the task into smaller steps
    • Fine-tuning a model on internal data
