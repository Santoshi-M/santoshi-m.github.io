Recommender Systems Ranking Personalization
P1 Contextual Recommendations ▼

A contextual recommendation system for news pages, designed to use the user’s immediate reading context to surface more relevant related content.

Built by adapting the existing 3-layer recommendation stack so ranking could respond to the specific page a user was actively consuming.
Challenge

News Pages are high-intent surfaces: the user has already chosen a specific story and expects follow-up recommendations to stay closely related. Previously, these pages reused the generic homepage ranking setup, so the system did not account for the current page context and often missed the chance to deepen the session.

  • The existing ranking flow ignored the seed page the user was currently reading.
  • The goal was to improve thematic relevance without rebuilding the recommendation system from scratch.
  • The team also needed a better way to measure “relatedness,” not just engagement.
Approach
  • Candidate generation: expanded the pool with contextual embedding retrieval, topic-based matching, and item-to-item collaborative signals.
  • Scoring: trained a dedicated model on News Page click data instead of relying on homepage behavior alone.
  • Reranking: prioritized relevance and contextual alignment, with lighter diversity constraints for tighter topical consistency.
  • Infrastructure: partnered with Backend to pass pageid into the online ranking stack and support the required feature fetches.
Engineering Trade-offs

The main trade-off was balancing immediate context with long-term user preference. We deliberately pushed the system toward stronger page-level relatedness while tuning the final reranking layer to avoid losing broader personalization quality.

Impact
  • Offline AUC improved by an absolute 5%.
  • DAU increased by more than 3%.
  • CPV increased by more than 10%.
  • Relevance improved by more than 20%.
Other Considerations

Beyond the model gains, it was also key to work with Product Management to define Relatedness as a core metric and build supporting dashboards, giving the team a clearer real-time view of contextual quality.

P2 Personalizing Publisher Recommendations at Scale ▼

A technical deep dive into evolving a static publisher widget into a personalized recommendation pipeline with reusable infrastructure, lightweight filtering, and scalable ranking.

Built as a fast MVP by reusing the existing article-ranking stack and adapting it for publisher discovery.
Challenge

The Suggested Publishers widget was a static list of a few highly regarded news and content publishers for each market. That meant, all the users in a particular market would see the same list of publishers irrespective of their preferences, ordered identically. As we know, relevance is the primary driver of Retention (LTV) and Daily Active Users (DAU), thus this was a missed opportunity to take advantage of the deep user understanding signals we have learned for the warm users in the ecosystem and generate personalised recommendations to increase user satisfaction.

  • Stagnant Engagement: Rule-based systems limits discovery, while an intelligent system surfaces niche publishers, increasing the coverage of the entire publisher pool.
  • Top-of-Funnel Conversion: Relevant interests lead to feed views, which are the primary drivers of long-term satisfaction.
  • The goal was to improve discovery quality without building a new system from scratch.
Approach
  • Candidate generation: map top-ranked articles back to their source publishers.
  • Filtering: remove already-followed, hidden, or blocked publishers.
  • Scoring: rank candidates using click, like, and long-dwell signals.
  • Reranking: balance user relevance with platform-level goals using soft boosts.
Engineering Trade-offs

The first version reused existing infrastructure to ship quickly. This reduced implementation cost and operational overhead, while accepting some bias inherited from the original recommender.

Impact
  • Delivered an MVP in roughly one week.
  • Improved click-through rate on discovery surfaces.
  • Increased exposure and engagement for less prominent publishers.
Next Step

A future iteration could use embedding-based retrieval and learned rankers to better capture semantic similarity between user intent and publisher content.

P3 Dynamic Button Placement ▼

A machine learning-driven UX project that dynamically placed the Continue Reading button on Article Pages based on real-time user behavior.

Shifted the experience from fixed, rule-based placement to intelligent behavior-based timing optimized for engagement and latency.
Challenge

The Continue Reading button is a critical engagement lever on Article Pages. Previously, it appeared at the same static position for every user, which often felt either too early or too late. At large scale, even small UX friction can reduce DAU and ad revenue, so the goal was to make placement adaptive without disrupting the reading flow.

  • The static experience treated all readers the same despite different reading patterns and intent levels.
  • The system needed to personalize button placement without introducing visible UI lag.
  • The project began with a clear objective but no concrete roadmap, so both product and technical direction had to be defined from scratch.
Approach
  • ML pipeline: built an end-to-end lifecycle covering session data collection, model training, and live inference.
  • Real-time decisioning: predicted the optimal button placement from in-session reading behavior fast enough for production UX constraints.
  • Cross-functional alignment: wrote the technical and product specifications to unblock execution and keep frontend and backend teams aligned.
Engineering Trade-offs

The main trade-off was latency versus accuracy. The model needed enough signal to make useful placement decisions, but inference also had to return within milliseconds. A slower model would cause the button to appear late, creating a worse experience than the original static version, so we prioritized a lightweight, high-throughput serving pipeline that could scale to million scale DAU traffic.

Impact
  • Moved the project from a vague concept to a live A/B test and production-ready implementation.
  • The experiment showed a statistically significant increase in DAU.
  • Replaced rigid manual placement rules with an automated system that adapts to millions of unique user reading patterns.
Key Lessons

The project reinforced that ownership often extends beyond code: documenting assumptions early, aligning teams at the spec stage, and maintaining end-to-end visibility across data, modeling, and inference were all essential for shipping a performant ML-powered UX feature.
